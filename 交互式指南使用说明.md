## 📋 6 步流程说明

### 第 1 步：准备工作
- ✅ 确认所有代码修改已完成
- ✅ 确保代码可以正常运行
- ✅ 记录本次修改的主要内容
- ✅ 准备好提交信息

### 第 2 步：本地测试与验证
- 🧪 编译 Fortran 代码（如有修改）
- 🧪 运行 Python 代码（如有修改）
- 🧪 运行框架检查脚本
- ✅ 确认所有测试通过

### 第 3 步：Git 提交本地仓库
- 💾 使用 `git status` 查看修改
- 💾 使用 `git add .` 添加文件
- 💾 使用 `git commit -m "信息"` 提交
- ✅ 遵循提交信息规范

### 第 4 步：创建本地时间戳备份
- 📦 运行自动备份脚本
- 📦 检查备份文件夹
- ✅ 确认所有重要文件已备份

### 第 5 步：推送到 GitHub 云端
- ☁️ 确认远程仓库配置
- ☁️ 使用 `git push` 推送
- ✅ 访问 GitHub 确认更新

### 第 6 步：完成确认
- ✅ 完成最终检查清单
- ✅ 验证所有步骤已完成
- 📝 规划下一步工作

## 🔧 自定义修改

如果您想修改指南内容，可以：

### 修改提交信息规范

编辑 HTML 文件中的 `step3` 部分，修改提交信息类型：

```html
<div>feat: 新功能</div>
<div>fix: 修复 bug</div>
<!-- 添加或修改您自己的规范 -->
```

### 修改检查清单

编辑对应步骤的 checklist，添加或删减检查项：

```html
<li onclick="toggleCheck(this)">
    <input type="checkbox">
    <span>您的检查项</span>
</li>
```

### 修改命令示例

编辑 code-block 部分，替换为您的常用命令：

```html
<div class="code-block">
<span class="comment"># 您的注释</span>
<span class="command">您的命令</span>
</div>
```
