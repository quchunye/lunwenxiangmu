## 📋 推荐方案

### 方案 1: Markdown 文档（推荐）⭐

**优点**:
- ✅ 简洁清晰，易于阅读
- ✅ 支持代码块和格式
- ✅ 可以用 Git 管理版本
- ✅ 方便搜索和索引

**实现方式**:
```markdown
# 对话主题 - 日期

## 任务目标
- 目标 1
- 目标 2

## 执行内容
### 1. 具体任务
- 操作：xxx
- 结果：✅ 成功 / ❌ 失败

### 2. 具体任务
- 操作：xxx
- 结果：✅ 成功 / ❌ 失败

## 关键成果
- 成果 1
- 成果 2

## 下一步
- [ ] 待办事项
```

---

### 方案 2: JSON 格式（结构化）

**优点**:
- ✅ 结构化数据
- ✅ 易于程序读取
- ✅ 可以自动化处理

**示例**:
```json
{
  "date": "2026-03-01",
  "topic": "文献阅读与备份更新",
  "tasks": [
    {
      "name": "阅读文献",
      "status": "success",
      "details": "阅读 3 篇 PDF"
    }
  ],
  "files_created": ["file1.md", "file2.md"],
  "git_commits": ["commit1", "commit2"]
}
```

---

### 方案 3: SQLite 数据库（高级）

**优点**:

- ✅ 可以建立索引
- ✅ 适合大量记录

## 💡 最佳实践

### 记录频率
- **每次重要任务后** - 记录关键决策和结果
- **每天结束时** - 简单总结当天进展
- **每周回顾** - 整理和归档

### 记录内容
- ✅ 任务目标
- ✅ 执行步骤（简要）
- ✅ 成功/失败状态
- ✅ 创建的文件
- ✅ Git 提交哈希
- ⚠️ 遇到的问题和解决方案

### 不需要记录的
- ❌ 详细的代码实现（已有 Git）
- ❌ 冗长的对话内容（只记要点）
- ❌ 临时性操作（不影响结果的）
