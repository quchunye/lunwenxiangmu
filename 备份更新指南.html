准备工作

📋 确认修改已完成

在开始备份之前，请确保您的大型修改已经完成并且代码可以正常运行。

  • 所有代码修改已完成
  • 代码已经过测试，可以正常运行
  • 已记录本次修改的主要内容
  • 准备好提交信息（描述本次修改）
💡 提示：建议您在开始之前先记录一下本次修改的主要内容，这样在提交时可以写一个清晰的提交信息。

本地测试与验证

🧪 运行测试确保代码正常

在提交之前，确保所有代码都能正常运行。

# 1. 切换到项目目录 cd "D:\Trae CN\论文" # 2. 测试 Fortran 代码（如果有修改） gfortran src\fortran\your_code.f90 -o test.exe .\test.exe # 3. 测试 Python 代码（如果有修改） python src\python\your_script.py # 4. 运行框架检查 .\check_framework.bat
  • Fortran 代码编译成功
  • Fortran 代码运行结果正确
  • Python 代码运行成功
  • 框架检查通过
⚠️ 注意：如果任何测试失败，请先修复问题再继续备份流程。

Git 提交本地仓库

💾 将修改提交到本地 Git 仓库

使用 Git 记录您的修改，这是版本控制的重要步骤。

# 1. 查看当前状态（查看哪些文件被修改） git status # 2. 添加所有修改的文件 git add . # 3. 查看添加的文件 git status # 4. 提交修改（替换下面的提交信息） git commit -m "feat: 添加您的修改描述" # 示例提交信息： # "feat: 实现不可约构型生成模块" # "fix: 修复超胞构建中的 bug" # "docs: 更新实验记录"
提交信息规范：
feat: 新功能
fix: 修复 bug
docs: 文档更新
style: 代码格式（不影响代码运行）
refactor: 重构
test: 测试相关
chore: 构建/工具配置
  • 已运行 git status 查看修改
  • 已运行 git add . 添加文件
  • 已运行 git commit -m "信息" 提交
  • 提交成功，看到提交哈希值
💡 提示：提交信息应该清晰描述本次修改的内容，方便日后查阅。

创建本地时间戳备份

📦 运行自动备份脚本

创建带时间戳的本地备份，确保可以回退到当前状态。

# 1. 运行自动备份脚本 cd "D:\Trae CN\论文" python scripts\backup\auto_backup.py # 或者使用批处理 .\scripts\backup\run_backup.bat # 输出示例： # ✓ 目录：src # ✓ 目录：docs # ✓ 文件：README.md # 备份完成：backups\timestamped\backup_20260301_203000
  • 已运行备份脚本
  • 看到"备份完成"提示
  • 检查 backups/timestamped/ 下有新的备份文件夹
  • 备份包含了所有重要文件
💡 提示：备份脚本会自动保留最近 30 天的备份，旧备份会被清理。

推送到 GitHub 云端

☁️ 将代码上传到 GitHub 仓库

将本地提交推送到远程仓库，实现云端备份和版本管理。

# 1. 确保在项目目录 cd "D:\Trae CN\论文" # 2. 查看远程仓库配置 git remote -v # 应该显示：origin https://github.com/quchunye/lunwenxiangmu.git # 3. 推送到 GitHub git push # 首次推送可能需要认证： # Username: quchunye # Password: 输入您的密码或 Token # 推送成功示例： # Enumerating objects: 10, done. # To https://github.com/quchunye/lunwenxiangmu.git # * [new branch] master -> master
  • 已确认远程仓库配置正确
  • 已运行 git push
  • 推送成功，看到成功信息
  • 访问 GitHub 仓库页面确认代码已更新
💡 提示：访问 https://github.com/quchunye/lunwenxiangmu 查看您的代码是否已更新。
